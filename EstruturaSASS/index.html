  • 2-layout
    • content_nome.scss Todos os arquivos de estilização do site vão estar na pasta layout e com nomenclatura parecida com este exemplo, onde cada página tem seu arquivo de estilização para uma melhor organização.
    • layout.scss O arquivo layout.scss faz a junção de todos os arquivos, todos os arquivos gerados de cada página é juntando em um só. Ex: @use 'content_nome';
  • 3-modules
    • _button.scss O arquivo _button.scss é um exemplo de vários outros, é um arquivo de módulo onde sua estilização é usada várias vezes. Este é um exemplo onde o arquivo vai ter a estilização de "todos os botões iguais" do site.
    • _form.scss O _form.scss é outro exemplo onde ele vai ter a estilização de "todos os formulários iguais" do site.
    • modules.scss O arquivo module.scss tem a função de juntar todos os módulos criados em um único arquivo.Ex: @use 'button'; @use 'form';
  • Sem Pasta
    • base.scss O arquivo base.scss tem a função de juntar a base de todas as páginas, isso para que depois que processar o sass ele junte todos os codigos nesse arquivo base e gere o arquivo style.css. Ex: @use '1-base/base.scss'; @use '2-layouts/layout.scss'; @use '3-modules/modules.scss';
    • style.css (criado automático) O style.css é gerado automático quando executado o código de processamento do sass, pegando o arquivo base.scss que nele está chamando todos os demais códigos sass e convertendo o código sass em css.
    • style.css.map (criado automático) O arquivo style.css.map é criado de forma automática e serve como um mapeamento para poder editar e em seguida visualizar as alterações feitas em sass sem ter que atualizar o DevTools.
